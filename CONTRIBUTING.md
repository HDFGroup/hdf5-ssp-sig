# Contributing to the SSP SIG

Thank you for your interest in helping improve the safety, security, and
privacy of the HDF5 ecosystem.

## 1. Before You Start

- Please read:
  - [CHARTER.md](./CHARTER.md)
  - [GOVERNANCE.md](./GOVERNANCE.md)
  - [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)
  - [SECURITY.md](./SECURITY.md)

## 2. Ways to Contribute

- **Propose work** using the "SSP Proposal" issue template.
- **Participate in meetings**, reviews, and discussions.
- **Author or review guidance**:
  - Checklists, hardening guides, sample configs.
  - Threat models and audit procedures.
- **Help with tests and CI**:
  - Improve governance “tests” in `tests/`.
  - Add linting, validation, and automation.

## 3. Workflow

1. Open an issue (proposal, question, or suggestion).
2. Discuss and refine the idea.
3. If appropriate, prepare a pull request.
4. Request review from relevant SIG members and editors.
5. Address feedback; once approved, the change will be merged.

## 4. Style & Conventions

- Prefer clear, actionable language.
- Use inclusive, respectful wording.
- Use Markdown headings consistently.
- Keep security-sensitive details out of public issues:
  - Use the channels defined in `SECURITY.md` for vulnerability reports.

## 5. License

By contributing, you agree that your contributions will be licensed under
the repository’s stated license(s).
